# Support

## Getting help

If you need help using Omnipod:

1. Read the [`README.md`](README.md) first.
2. Check the troubleshooting section.
3. Open a GitHub issue using the appropriate template.

## What to include

Include as much of the following as possible:

- operating system and version
- shell / terminal used
- Docker version
- install method (`install.sh` or `install.ps1`)
- Omnipod commit or release
- exact command used
- relevant logs with secrets removed

## What not to post publicly

Do not publish:

- provider API keys
- `.env` contents
- bearer tokens
- cookies / session data
- internal hostnames or private URLs

## Security issues

For vulnerabilities, do **not** open a public issue.
Use the guidance in [`SECURITY.md`](SECURITY.md).
