# Support

## Getting help

1. Check the README troubleshooting section first.
2. If it's not covered there, open a GitHub issue with the bug report template.

## What to include

- OS and version
- Shell or terminal
- Docker version
- Whether you used `install.sh` or `install.ps1`
- Omnipod version or commit
- The exact command you ran
- Relevant logs, with secrets removed

## What not to post publicly

- Provider API keys
- `.env` file contents
- Bearer tokens
- Cookies or session data
- Internal hostnames or private URLs

## Security issues

Use the guidance in [SECURITY.md](SECURITY.md). Don't open a public issue for vulnerabilities.
